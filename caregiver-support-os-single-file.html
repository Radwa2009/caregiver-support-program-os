Staff toolkit · demo panel

AI assistant

Copy-ready prompt templates staff can paste into Claude. This panel is a demo — it does not call an API. Every template carries the same guardrails.

Guardrails built into every prompt
  • Use de-identified data only (case IDs and initials, never names or contact details).
  • Do not make medical, legal, financial, or eligibility decisions.
  • Do not invent facts; flag uncertainty and missing information clearly.
  • Final decisions stay with trained humans.
  • Verify all translations with a qualified speaker before sending.

Sustainability

Documentation & handoff

Written so the tool can outlive any single staff member. This is the page a new coordinator reads first.

How staff should use this tool

  1. Log every new referral through the Referral intake form the same day it arrives.
  2. Check the Dashboard each morning for follow-ups due and human-review flags.
  3. After each workshop, log it in the Workshop tracker and note any follow-up requests.
  4. Use the AI assistant templates for drafting only — every AI output is reviewed and edited by a trained staff member before it reaches a caregiver.
  5. On the first business day of each month, generate the Monthly report, review it, and add coordinator commentary before sharing.

What data should not be entered

  • No full names, addresses, phone numbers, emails, or dates of birth.
  • No medical record numbers, diagnoses, medications, or clinical details.
  • No financial account details or benefits case numbers.
  • If a note needs identifying detail to be useful, it belongs in the organization's approved system of record — not here.

Privacy and consent guidelines

  • Confirm consent before creating a case; the intake form will not save without it.
  • Use case IDs in all AI prompts, reports, and shared documents.
  • Follow the organization's privacy policy and data retention schedule; this demo stores data only in the local browser.
  • When in doubt about what to record, ask the program manager before entering it.

How to train a new staff member (about 1 hour)

  1. Read this page together (10 min).
  2. Walk through one fictional intake end-to-end (15 min).
  3. Practice filtering the caregiver list and updating a status (10 min).
  4. Review the AI guardrails and try one prompt template with demo data (15 min).
  5. Generate a monthly report and discuss what a coordinator would edit (10 min).

How to maintain the workflow

  • Weekly: clear or reschedule overdue follow-ups; review all human-review flags with the program lead.
  • Monthly: generate the report, archive a copy per policy, and review whether needs categories still match what caregivers ask for.
  • Quarterly: revisit prompt templates and privacy guidance with staff; retire anything unused.

How to run a 2-week pilot

  1. Week 0: pick 2–3 staff testers, agree on the de-identification rules, and reset the demo data.
  2. Week 1: log all new referrals and one workshop in the tool alongside the current process (double entry, on purpose).
  3. Week 2: run a morning check-in from the dashboard daily; draft one follow-up message using an AI template with staff review.
  4. Close-out: 30-minute debrief — what was faster, what was missing, what felt risky — and decide go / no-go on a production build.

Pilot success metrics

  • 100% of pilot referrals logged with consent documented and no identifying data entered.
  • Follow-ups due are visible each morning without opening a spreadsheet (target: under 2 minutes to answer "who needs follow-up today?").
  • Monthly report drafted in under 30 minutes, versus a multi-hour manual process.
  • Every AI-drafted message shows evidence of human editing before send.
  • Staff confidence using the tool rated 4+ out of 5 at debrief.

About this project

Built by Radwa Gebreil

Public Health & Nonprofit AI Workflow Builder · MPH, UC Berkeley · BA Communication, UC Davis

This prototype is inspired by caregiver support, older adult support, program evaluation, surveys, dashboards, reporting, and a public health practicum focused on program development and evaluation for caregivers, older adults, and individuals needing care coordination.

Why this exists

Caregiver support programs often run on scattered notes, spreadsheets, and emails. This makes it hard to answer three simple questions: who needs follow-up, what do caregivers need most, and is the program working? This prototype puts those answers on one screen.

How it was built

  • I built this prototype with Claude as a product-thinking and coding partner.
  • I designed the workflow based on public health, caregiver support, and nonprofit program evaluation experience.
  • I would keep medical, legal, financial, eligibility, and service decisions with trained humans — this tool organizes information; people decide.

Designed as a Claude Corps portfolio project

The Claude Corps Fellowship values people who can enter mission-driven organizations, understand messy workflows, build practical AI-assisted tools, train non-technical staff, and leave behind sustainable systems. Each section of this app maps to one of those skills — the intake and dashboard to workflow understanding, the AI panel to safe AI adoption, and the documentation page to training and handoff.
